# Syntax Field Guide

## Concept

A lay-person-forward field guide that helps a first-time Syntax user move from an empty GitHub repository to a tested, editable, publishable small website.

## Audience

Solo builders, designers, writers, and beginners who are comfortable following instructions but do not want to become framework maintainers before publishing a site.

## Problem

The framework documentation explains Syntax itself, but first-time users need a practical walkthrough of the complete workflow, including Terminal basics, setup prompts, testing, local ports, Git, GitHub Pages, and common errors.

## Primary action

Start the walkthrough.

## Tone and direction

Editorial, calm, plainspoken, and beginner-friendly. The page should feel like a field guide rather than API documentation.

## Recipe and features

- Recipe: Blank
- Visual direction: Editorial
- JavaScript features: None required
- Deployment: GitHub Pages
- Mode: Ship

## Scope

- Explain what Syntax solves and where it fits.
- Teach the factory/project mental model.
- Document the empty-repository and sibling-folder workflow.
- Explain setup prompts and commands.
- Cover testing, previewing, committing, pushing, and publishing.
- Include real troubleshooting lessons from the Olive dogfood project.

## Non-goals

- Do not become a complete Git, Terminal, Node.js, or GitHub course.
- Do not duplicate the Component Lab or framework API documentation.
- Do not require client-side rendering or unnecessary JavaScript.
